Attribute Value Information
Is WordPress Not checked Determines if the site is built with WordPress. This is important for applying WordPress-specific security checks.
WordPress Version Not checked Checks the WordPress version. Keeping WordPress updated is crucial for security. Visible versions may expose vulnerabilities.
Theme Not checked Identifies the active theme. Themes should be kept updated to patch security vulnerabilities.
Plugins Not checked Lists detected plugins. Like themes, plugins should be kept updated to maintain security.
SSL Enabled Not checked Checks if the site uses HTTPS. SSL encryption is crucial for securing data transmission between the server and users.
Directory Indexing Not checked Checks if directory listing is enabled. If enabled, it can expose sensitive files and information to potential attackers.
wp-cron.php Found Not checked Checks if wp-cron.php is accessible. If found, it could potentially be used for Denial of Service attacks.
User Enumeration Not checked Checks if user enumeration is possible. This can be used by attackers to gather usernames for brute force attacks.
XML-RPC Enabled Not checked Checks if XML-RPC is enabled. While useful for some integrations, it can be exploited for brute force attacks if not properly secured.
Hosting Provider Not checked Identifies the hosting provider. Different providers offer varying levels of security and performance.
Issues Not checked Summarizes potential security issues found during the scan.
